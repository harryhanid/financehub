1. Executive Summary / Ringkasan Eksekutif

Finance Hub adalah sistem manajemen keuangan internal berbasis web untuk Account Payable, Petty Cash, Beasiswa, dan proses pembayaran yang dijalankan secara lokal (localhost/LAN). Dirancang sebagai platform multi-perusahaan untuk Sinar Mas Tjipta dan Eka Tjipta Foundation.

Finance Hub menggantikan sistem lama berbasis file Microsoft Excel VBA Macro (DbBeasiswa v2.2.xlsm) yang dioperasikan via shared folder Y:/. Lihat perbandingan lengkap di § 2.5.

2. Problem Statement / Latar Belakang & Tujuan

2.1 Masalah Sebelumnya

Sebelum Finance Hub, pengelolaan keuangan beasiswa dilakukan dengan file Excel VBA Macro (DbBeasiswa v2.2.xlsm) via shared folder Y:/:

  • File lock conflict — hanya 1 orang bisa edit; yang lain Read-only.
  • Tidak ada autentikasi — siapapun yang akses Y:/ dapat membuka, mengubah, bahkan menghapus data.
  • Tidak ada audit trail — tidak diketahui siapa yang mengubah data dan kapan.
  • Pembuatan Payment Approval Memo dilakukan manual dengan mengisi sheet Detail PAM dan Rangkuman PAM, lalu mencetak.
  • PAM Recon menggunakan PivotTable Excel yang harus di-refresh manual setiap ada perubahan data.
  • Days of PAM (4 tanggal SLA) dicatat sebagai kolom Cat3–Cat7 di sheet DbPayment, tanpa validasi format tanggal.
  • Data Medical/Hospital (DbHospital) dikelola dalam sheet terpisah, tidak terintegrasi otomatis dengan rekap kategori.
  • Data SMT dan ETF tidak terpisah secara sistematis.

2.2 Solusi

4. User Journey Flows / Alur Pengguna

4.1 Login & Company Selection

Buka http://localhost:8080 → Redirect ke /login (jika belum punya token) → Input username + password → Verifikasi JWT → redirect ke /select-company → Pilih perusahaan: [Sinar Mas Tjipta] | [Eka Tjipta Foundation] → Masuk ke Dashboard perusahaan yang dipilih

4.2 Alur Input Beasiswa (ETF)

Dashboard ETF → Menu: Beasiswa Tab: Data Siswa → Cari (nama/kode/jenjang/angkatan/program/univ/fak) + Filter (Program/Pillar/Status) → Tambah Siswa Baru (auto-generate kode, isi IPK) → Lihat kolom Budget / Payment / Sisa per siswa Tab: Input Budget → Pilih siswa via autocomplete → Isi: Tanggal, Pillar, batch item (Cat1 / Cat2 / Amount) → Simpan Tab: Data Budget → Filter: Kata Kunci / Kategori 1 / Pillar / Bulan / Tahun / Program → ★ Summary badges mencerminkan filter aktif (bukan total global) → Edit / Hapus baris · Export CSV | Export PDF Tab: Input Payment → Header: Tanggal, PAM No (auto-format PAM-XXX-ETF-MM-YYYY), Pillar, Perusahaan → "+ Tambah Baris": cari siswa, pilih Kat1/Kat2, isi Amount, isi 4 tanggal SLA → "Simpan" → baris → payment_beasiswa (draft) + PAM record otomatis dibuat Tab: Data Payment → Filter: Kata Kunci / Kategori 1 / Pillar / Bulan / Tahun / Program / Status → ★ Summary badges mencerminkan filter aktif (bukan total global) → Export CSV | Export PDF Tab: Report Siswa → Pilih siswa via autocomplete → data lengkap + IPK + rekap + detail → Export CSV | Export PDF

4.3 Alur Payment Approval Memo (PAM)

Menu: Payment Approval Memo Tab: Daftar PAM → Tabel semua PAM records (auto-created saat Input Payment) → Isi / update GL Account per PAM via dropdown COA Tab: Draft Memo → Cari PAM No via autocomplete → Preview form memo: No PAM, tanggal, GL Account, Cost Center → Edit inline: nama penerima, rekening, catatan, jumlah → Lampiran: detail per siswa (collapsible) → Export PDF (format standar) | Export Excel (2 sheet: Rangkuman + Detail) Tab: Days of PAM → Search PAM No atau Nama Siswa (search-first, lazy-load via AJAX) → Filter: Bulan, Tahun, Status SLA → Pilih baris via checkbox → Bulk update 4 tanggal SLA sekaligus

8.2 Alur PAM End-to-End

1. User input payment di Beasiswa → klik "Simpan" 2. Service layer (atomic transaction): a. Simpan baris ke payment_beasiswa (status: draft) b. Auto-create PAM record di pam_records (nomor PAM dari field input, company_id, total amount, tanggal) 3. Tab "Daftar PAM": - PAM muncul di tabel: PAM No, Tanggal, Total, GL Account - User pilih GL Account dari dropdown COA 4. Tab "Draft Memo": - Cari PAM No → form preview otomatis terisi - Edit inline nama, catatan, jumlah - Lampiran detail siswa (collapsible) - Export PDF atau Export Excel 5. Tab "Days of PAM": - Search PAM No / Nama Siswa (lazy-load) - Centang baris → Bulk update 4 tanggal SLA

11. Security & Access Control

11.1 Authentication Flow

1. POST /api/auth/login → verifikasi password bcrypt 2. Generate access_token (JWT, 1 jam) + refresh_token (JWT, 7 hari) 3. Setiap request → middleware decode access_token 4. Token expire → gunakan refresh_token 5. Logout → revoke refresh_token di database

11.2 Security Controls
